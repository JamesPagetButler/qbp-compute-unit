Summary
-------
Implemented the cross-repo compatibility canary specified in
doc/wyrd-integration.md §5.1. Workflow checks out this repo and Wyrd
into sibling directories, pins Wyrd's emulator import via `go mod edit
-replace` to the local checkout, then runs `go test -race -count=1
./compute/...` followed by a `go build ./...` sanity step.

Enhancements over the minimal §5.1 template:
  - Header comment block: purpose, spec ref, #15 auth gating note.
  - permissions: contents: read (least-privilege).
  - actions/checkout@v4 + actions/setup-go@v5 (pinned major versions).
  - go-version-file: wyrd/go.mod (Go version follows Wyrd's go.mod).
  - GOFLAGS: -mod=mod env to allow local replace without go.sum churn.
  - go mod tidy || true after replace (best-effort, non-fatal).
  - go test invoked with -race -count=1 per go-coding-guide.md.
  - Sanity `go build ./...` step after tests to catch link-time breakage.

Auth gating
-----------
Wyrd is private as of 2026-05-07. The canary uses
`${{ secrets.WYRD_PAT }}` for the cross-repo checkout. Until issue #15
resolves (option A: make both repos public, OR option B: provision
WYRD_PAT as a repo secret), this canary will fail at the Wyrd checkout
step with an auth error. The YAML structure is correct; the secret is
the only blocker. When #15 lands, the canary starts passing without
any YAML changes.
