AC2 NOTE (REVIEWER PLEASE READ)
==============================================================================

The required AC2 grep pattern is:
    git ls-tree -r HEAD --name-only | grep -E '\.test$|\.wasm$|cth/bin/|hw-eval'

The fourth alternative — "hw-eval" with no "$" anchor — matches any tracked
path that contains the substring "hw-eval", including the directory path
"cmd/hw-eval/...". The repository has 3 legitimate tracked source files in
that directory:

    cmd/hw-eval/go.mod
    cmd/hw-eval/main.go
    cmd/hw-eval/walk_options_report.md

The AC2 grep therefore is NOT empty after my commit. The 4 binary leak files
ARE removed (binary "cmd/hw-eval/hw-eval" is gone; the regex match is on the
directory component "hw-eval/", not the binary). I have deliberately NOT
deleted those legitimate source files — they are out of scope per the
SCOPE clause.

A more precise grep that captures the spec intent (binary files only) is:
    grep -E '\.test$|\.wasm$|cth/bin/cth$|cmd/hw-eval/hw-eval$'
which IS empty after this commit (verified — see "AC2 — precise variant"
section below).

I ran the AC2 grep verbatim as written and report its true (non-empty)
output. Reviewer to interpret.
