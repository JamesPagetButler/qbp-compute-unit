## 1. The Lesson of Squam: Data is Scale-Relative
The Squam Lake loon investigation (Test 12-B) revealed that a high-precision model (QW256) is easily "bogged" if global data (e.g., hurricanes) is treated with the same algebraic status as local data (e.g., loon census). We formalize the solution as **Reciprocal Focus**.

## 2. Subject-Background Duality
Every Worktree investigation must define its **Subject Locales**.
*   **Subject (The Focal Zone):** Data points within the investigation's causal chain. These are loaded at full precision (QW128/256).
*   **Background (The Reciprocal):** Everything else in the hypergraph. This data is not deleted, but its **Algebraic Weight** is set to zero ($0.0$) in the current frame.

## 3. Lossless Dismissal (The QROT Filter)
We utilize the Persona's Stance (1024-bit Rotation) to perform dismissal:
*   **Rotation to Imaginary:** Data that is orthogonal to the Persona's focus (e.g., lead levels during a hurricane track) is rotated entirely onto the imaginary axis.
*   **Identity Contribution:** Because it is imaginary in the current frame, it contributes $0.0$ to the real-world flux, momentum, or gradient calculations.
*   **Lossless Integrity:** The data is never "dropped" (lossy); it is merely "rotated out of focus." If the Persona shifts stance, the data can be rotated back into the real axis instantly.

## 4. Promotion via Connective Tissue
Data is only promoted from Background to Subject if a **"Seam"** is detected.
*   **Intersection Trigger:** If a Background node's world-line physically or algebraically intersects a Subject node (e.g., rainfall from a hurricane entering the watershed), the **Information Cart** automatically promotes the intersecting nodes to the focal precision.
