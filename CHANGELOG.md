# Changelog

All notable changes to the QBP Compute Unit emulator module are documented here.

The format is loosely based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/);
this project does **not** follow strict SemVer at sub-`v1.0.0` versions — the `Xqbp*` ISA
surface evolves during the Crawl/Walk phases per `architecture/peer-review-005-stream-migration.md`.
Tag-level stability promises start at `emulator/v0.1.0` (post-Walk smoke testing) per
issue [#20](https://github.com/JamesPagetButler/qbp-compute-unit/issues/20).

## [emulator/v0.1.0-rc1] — Wyrd-integration-ready release candidate

**Status:** release candidate. Stable enough to integrate against (Wyrd, BMA, Contextus
consumers may pin to this tag); **not** yet stable enough to depend on long-term. The
final `emulator/v0.1.0` tag will follow after Walk-phase smoke testing per issue
[#20](https://github.com/JamesPagetButler/qbp-compute-unit/issues/20) gates.

This is the first cut of the M0 cohort (Authority Chain Hardening per
`architecture/peer-review-005-stream-migration.md` §4 M0). Consumers should follow
`doc/wyrd-integration.md` v0.2 for the public API surface.

### Added

- **`emulator/public_api.go` — typed-per-width Gearbox surface** ([#23](https://github.com/JamesPagetButler/qbp-compute-unit/pull/23), closes [#10 D1](https://github.com/JamesPagetButler/qbp-compute-unit/issues/10)).
  18 methods on `*Gearbox` exposing the Crawl-phase consumer surface:
  - Quaternion ℍ at QW64: `QMul64`, `QAdd64`, `QRot64`, `QConj64`, `QNorm64` (`[4]float64`)
  - Quaternion ℍ at QW128 (double-double): `QMul128`, `QAdd128`, `QRot128`, `QConj128`, `QNorm128` (`[8]float64`, hi×4 then lo×4)
  - High-precision software fallback: `QMulHighPrec(w Width, a, b [4]float64) ([4]float64, error)` for W256+
  - Complex ℂ: `CMul64`, `CAdd64` (`[2]float64`); `CMul128` (`[4]float64`)
  - Octonion 𝕆: `OMul64`, `OAdd64` (`[8]float64`) — returns `ErrTierUnsupported` in Crawl; activates with the `Xqbpoct` extension at M1+
  - Sedenion 𝕊: `SMul64`, `SAdd64` (`[16]float64`) — returns `ErrTierUnsupported` in Crawl; activates with the `Xqbpqec` ZDCHK extension at M2+
  - `SedenionResult` struct exposes `Value` + `ZDClass` + `ZDIndices` per `Archive/RV-Fano-Implementation-Refinements.md` §2
  - `ErrTierUnsupported` sentinel error
  - 9 hot-path methods verified at `0 B/op, 0 allocs/op` on FX-8350

- **`lean2rom` Lean→ROM authority-chain pipeline** ([#12](https://github.com/JamesPagetButler/qbp-compute-unit/pull/12), closes [#7](https://github.com/JamesPagetButler/qbp-compute-unit/issues/7)).
  `cmd/lean2rom` derives sign and index ROMs from `lean/QBP/Sedenion.lean` (or a native Go Cayley-Dickson fallback). Emits `roms/{sedenion,octonion}_{signs,idx}.hex` + `roms/quaternion_signs.go` + `roms/CHECKSUMS.lean-verified`. `make verify-roms` enforces the chain in CI.

- **AVX-FMA QW64 + QW128 fast paths** ([#11](https://github.com/JamesPagetButler/qbp-compute-unit/pull/11), closes [#1](https://github.com/JamesPagetButler/qbp-compute-unit/issues/1) and [#8](https://github.com/JamesPagetButler/qbp-compute-unit/issues/8)).
  - `emulator/qmath_amd64.s`: AVX-FMA quaternion Hamilton product, conjugate, rotate, norm at QW64
  - `emulator/qmath_128_amd64.s`: double-double (Dekker/Bailey) Hamilton product at QW128
  - `emulator/qmath_scalar.go`: pure-Go scalar fallback
  - `emulator/wdevent.go`: WDEvent passive emission tap at the ISA boundary (M0.2). Buffered channel; non-blocking; `WatchdogDropCount` atomic counter
  - QW128 algebraic lifetime: 172 days @ 1 GHz composition

- **Lean-derived assembly constants** ([#24](https://github.com/JamesPagetButler/qbp-compute-unit/pull/24), closes [#13](https://github.com/JamesPagetButler/qbp-compute-unit/issues/13) and [#14](https://github.com/JamesPagetButler/qbp-compute-unit/issues/14)).
  Both `qmath_amd64.s` and `qmath_128_amd64.s` now consume the shared `qbp_lean_sign_x/y/z` and `qbp_lean_conj` symbols generated by `lean2rom` into `emulator/qmath_constants.s`. Authority chain closed at the asm level: `Lean → ROM hex → qmath_constants.s → kernel asm`. `TestSIMDConstantsMatchROM` parses the asm DATA blocks at test time and verifies byte-for-byte agreement with the ROM source-of-truth. Vendor-prefix conformant per `Ref/RISC-V-Policies-and-Best-Practices.md` §4.

- **`doc/wyrd-integration.md` v0.2 design doc** ([#21](https://github.com/JamesPagetButler/qbp-compute-unit/pull/21), closes [#10 D0](https://github.com/JamesPagetButler/qbp-compute-unit/issues/10)).
  Locked Q1/Q2/Q3 architectural decisions for the QBP-CU ↔ Wyrd integration: canonical import path, Gearbox/Accelerator surface separation, Tier ⊥ Width orthogonality.

- **Wyrd-compatibility CI canary** ([#22](https://github.com/JamesPagetButler/qbp-compute-unit/pull/22), closes [#10 D2](https://github.com/JamesPagetButler/qbp-compute-unit/issues/10)).

Detects breaking-change API drift before it hits Wyrd consumers. Currently auth-gated on `WYRD_PAT` per [#15](https://github.com/JamesPagetButler/qbp-compute-unit/issues/15); skips cleanly when unset.

### Changed

- `emulator/qmath_amd64.s`: hand-derived sign-mask constants (`y_sign_x/y/z`, `y_conj`) replaced with `qbp_lean_*` references to `emulator/qmath_constants.s` (#24). Pure symbol rename; instruction logic unchanged. Performance improved 5–10% across all ops (cache locality from RODATA consolidation).
- `emulator/qmath_128_amd64.s`: same rename as QW64; same performance improvement. Symbols shared with QW64 (sign-bit XOR is precision-independent).
- `cmd/lean2rom`: output path changed from `emulator/asm/qmath_constants.s` → `emulator/qmath_constants.s` to enable cross-file Go-assembler linkage (#24). Symbol prefix from `y_sign_*` to `qbp_lean_*` for vendor-prefix conformance.

- Repository hygiene ([#19](https://github.com/JamesPagetButler/qbp-compute-unit/pull/19), closes [#16](https://github.com/JamesPagetButler/qbp-compute-unit/issues/16)).

### Verification

This release candidate passes:

- `cd emulator && go test -race -count=1 ./...` — all tests green
- `cd emulator && go vet ./...` — clean
- `make verify-roms` — all 4 ROM checksums match Lean source
- `cd emulator && go test -bench=. -benchmem -count=10 -benchtime=3s ./...` — all benchmarks `0 B/op, 0 allocs/op`; medians within FX-8350 5% noise band (most ops 5–10% faster post-#24)

### Known gaps before `emulator/v0.1.0` final

Per `reviews/peer-review-006-wdevent-pr11-redteam.md` §7 Item 4: WDEvent emission overhead is 5–8% on QCONJ / QMUL128 / QADD128 vs pre-emission baseline. Above the original 5% AC. Carved out as an architecture/-side mitigation item for Walk-α; does not affect the Gearbox API surface Wyrd consumes.

Issue [#15](https://github.com/JamesPagetButler/qbp-compute-unit/issues/15) (`WYRD_PAT` / `QBP_PAT` cross-repo CI tokens) is open. Until resolved, the Wyrd-compatibility canary skips with a `::notice::` annotation rather than executing. Once provisioned, the canary activates automatically — no YAML changes required.

ADR-004 (M1 Gearbox state model) remains **Proposed**. M1 implementation (#26 follow-ups) will introduce the CSR-bound stateful surface + QW8 peripheral methods + goroutine-pair dispatch; consumers pinning to `v0.1.0-rc1` continue to compile against the additive Gearbox surface without modification.

### Implementation provenance

The M0 cohort was implemented via a multi-agent execution discipline: `general-purpose` Agent-tool sub-agents in isolated worktrees, with the `qbp-cu-implementor` session as reviewer (verbatim test/bench evidence required, race detector mandatory per `~/Documents/go-coding-guide.md`, scope-glob enforcement, agent-evidence archived to `reviews/agent-evidence/issue-N-attempt-M.txt`). One principled rejection on `#14` attempt-1 surfaced the asm-package-boundary finding that led to PR #24's path-(b) correct closure.
